Sobre

Sou estudante de Engenharia de Software na UniCuritiba com experiência prática como desenvolvedor Full Stack. Meu terreno é o ecossistema .NET — C#, ASP.NET Core, Blazor, Entity Framework — com PostgreSQL e SQL Server na camada de dados, e Docker e GitHub Actions cuidando do resto.

Meus melhores projetos nascem da interseção entre jogos e engenharia: editor de mapas para Victoria 3, bot de Discord para sessões multiplayer, rastreador de preços da Steam. Cada um me forçou a aprender algo real — de flood-fill em canvas a políticas de resiliência com Polly.

Hoje meu foco é consistência: commits diários, documentação decente e arquitetura que se sustenta — não só código que funciona.

Projetos em destaque

Steam Price Tracker v2 ↗ Em desenvolvimento

Problema
Preços na Steam mudam o tempo todo e não há histórico acessível para decidir a hora certa de comprar.
Solução
Worker service em .NET 9 que coleta preços da API da Steam com resiliência (Polly), persiste em PostgreSQL via EF Core e expõe tudo em uma API ASP.NET Core com frontend Blazor. Logging estruturado com Serilog, Docker e CI/CD com GitHub Actions.
Resultado
Evolução do v1 (serverless) para uma arquitetura de produção, construída em público com commits diários.

Atlas — Editor de Mapas para Victoria 3 Ferramenta

Problema
Editar províncias de Victoria 3 exige manipular arquivos e cores manualmente — lento e propenso a erros para modders.
Solução
Editor no navegador com pintura por flood-fill em canvas, exportação de definitions.csv compatível com o jogo e validador de mapas integrado. Zero backend, zero instalação.
Resultado
Fluxo de criação de mapas que levava horas passa a ser visual e imediato, direto no browser.
JavaScriptCanvas APIFlood-fillCSVModding

Bot Discord — Victoria 3 Multiplayer Em arquitetura

Problema
Organizar sessões multiplayer de grand strategy no Discord é caótico: quem joga com qual nação, quando, e como visualizar isso.
Solução
Bot em discord.js com comandos para gerenciar jogadores por nação, agenda de sessões e geração dinâmica de mapas coloridos conforme a distribuição de países.
Resultado
Arquitetura definida; renderização de mapas em desenvolvimento.

Robô Solucionador de Labirintos Acadêmico

Problema
Navegar um labirinto físico de forma autônoma com hardware de baixo custo e sensores instáveis.
Solução
Arduino + Adafruit Motor Shield + 3 sensores IR, com calibração para estabilizar leituras e integração serial com Node-RED para dashboard de telemetria em tempo real.
Resultado
Robô funcional do hardware ao gráfico — depuração de fiação de motores, instabilidade de sensores e comunicação serial resolvidas.
